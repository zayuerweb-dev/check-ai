关于 Check.AI

Check.AI 是一个 AI 平台与模型对比数据库 — 会员、API 价格、上下文窗口、能力、跑分。站点由一位独立开发者维护。这一页解释：是谁在做、数据怎么来、怎么保持准确。

维护者

Check.AI 由 GitHub 上的 @zayuerweb-dev 维护。整站源码（包括对比数据、页面模板、每日同步流水线）公开：

  • 仓库：github.com/zayuerweb-dev/check-ai
  • 问题反馈 / 数据修正：在仓库开 GitHub Issue 或 PR
  • 部署：Cloudflare Pages

看到错的数字、过期的价格、缺的模型 — 开 Issue，下次同步就修。

数据来源

每个页面背后是三类数据：

  • 价格与规格。每天从公开数据集 models.dev（社区维护的全平台模型快照）拉取，并对照各厂商官方价格页（OpenAI、Anthropic、Google、xAI、DeepSeek、阿里、Mistral）核对。
  • 跑分。来自公开榜单：SWE-bench Verified、LMArena、HumanEval、LiveCodeBench、MMLU-Pro、GPQA、AIME。对比页和 topic 页里的数字按整百分位四舍五入 — 因为不同评测批次之间会有正常波动。
  • 编辑判断。"适合谁"、"30 秒结论"、"人话总结"等段落由维护者基于真实使用经验撰写：生产代码、agent 工作流、内容流水线里跑过这些模型得出的判断。是观点不是事实，所以会标日期，方便你判断信息的新鲜度。

更新节奏

数据层和编辑层节奏不同：

  • 每天：GitHub Action 拉 models.dev，有变化就开 PR。价格、新模型、能力标签 24-48 小时内同步。
  • 每周：编辑判断和跑分对照新数据 review 一遍，过时的就重写。
  • 大版本：有重大模型发布（如 Claude 5、GPT-5.5）当周改动相关 topic 和对比页。

编辑标准

  • 带日期，不说 "最新"。每篇文章和 topic 都有发布日期和"最后更新"日期。AI 变化太快，不带日期的"最新"等于过期。
  • 有出处。价格和跑分链接到原始页面或榜单。估算或社区数据会标出来。
  • 中立。站点不接受任何模型厂商的赞助或合作。编辑判断只在具体维度上选优 — 不评"全场最佳"，因为答案永远依赖你的负载。
  • 可证伪。能拿出具体的 benchmark 或价格事实证明我们错了 → 我们就改。

利益冲突声明

  • Check.AI 没有付费赞助、不放展示广告。
  • 部分外链（例如 OpenRouter）带 utm_source 参数仅用于流量归因。OpenRouter 当前没有公开的推广返佣计划，所以这些链接没有任何佣金 — 是基于价值的真实推荐。
  • 维护者目前与所有模型厂商（OpenAI、Anthropic、Google、xAI、DeepSeek、阿里、Mistral）没有股权、雇佣或外包关系。
  • 未来如果接入真实 affiliate（例如 Together AI、Cursor 推广）会在链接处和这一页同时披露。

Check.AI 不是什么

  • 不能替代你在自己数据上的评测。
  • 不是 SLA / 实时 API 状态页 — 出故障请看厂商官方 status page。
  • 不是穷举 — 只深度跟踪 2026 年全球前沿八款主流模型。区域性 / 小模型按读者需求添加。

联系方式
