30 秒结论

  • 新档位：Fable 5 是 Anthropic 目前最强的"广泛发布"模型，开了一个排在 Opus 之上的 Mythos-class 档位。
  • 价格翻倍：$10 / $50（每 1M token，输入/输出），是 Opus 4.8（$5/$25）的两倍。1M 上下文、单次最高 128k 输出。Batch 价 $5/$25，缓存输入打 1 折。
  • Fable 5 vs Mythos 5 = 只差安全护栏：同一个模型、同样能力和价格。Fable 5（公开版）带安全分类器，遇到网络安全、生物化学、模型蒸馏等敏感请求会拒绝或转给 Opus 4.8 回答；Mythos 5（受限版）去掉了这些分类器，只通过 Project Glasswing 向授权机构开放。你和绝大多数开发者用到的是 Fable 5。
  • 谁该上：需要最强推理、长程 agent 的新项目可以上 Fable 5；日常、量大、成本敏感的场景，Opus 4.8（$5/$25）甚至 Sonnet 4.6（约 $2.6/$15）更划算。6/9–6/22 订阅计划免费试，值得趁这段试一把。
  • 跑分：官方只说"几乎所有基准 SOTA"，没有公布具体数字——本文不替它编。

已确认的事实

截至 2026 年 6 月，关于 Fable 5 / Mythos 5 可以确定的：

  • 发布：2026-06-09。
  • 价格：$10 / 1M 输入、$50 / 1M 输出，两个模型一致。官方说法是"不到 Mythos Preview 价格的一半"。Batch（异步）$5/$25，prompt cache 命中部分约 1 折。
  • 规格：1M token 上下文，单次最高 128k 输出 token。
  • API 名称：claude-fable-5 和 claude-mythos-5。
  • 可用平台：Fable 5 在 Claude API、AWS（含 Amazon Bedrock）、Google Vertex AI、Microsoft Foundry 全面可用；Mythos 5 不公开发售，只在 Project Glasswing 里向通过审核的客户开放。
  • 数据保留：两者都需要 30 天数据保留（属于 Covered Models），不支持零数据保留。
  • 思考模式：只有"自适应思考"一种模式且常开，不能关；原始思维链不会返回，只能拿到摘要或留空。

Fable 5 和 Mythos 5 差在哪

这是最容易被名字误导的地方。两个模型底层架构、能力、上下文、价格全部相同。区别只有一处：安全护栏。

  • Claude Fable 5（公开版）：内置安全分类器。当请求落在网络安全、生物/化学、模型蒸馏等敏感领域时，它可能拒绝，或者把这次请求转交给次强的 Claude Opus 4.8 来回答。这是面向所有人的通用版本。
  • Claude Mythos 5（受限版）：同一个模型，去掉了这些分类器。它通过 Project Glasswing（Anthropic 与美国政府合作的项目）限量开放，最初只给现有 Mythos Preview 用户——网络防御者、基础设施提供方等；后续计划在保留网络安全护栏的前提下，向生物研究者开放。要拿到访问权，得联系 Anthropic、AWS 或 Google Cloud 的客户团队。

一句话：Mythos 5 不是"更强的 Fable 5"，而是"放开了部分安全限制、给特定机构用的 Fable 5"。对普通开发者，二者能力没有差别，你接的就是 Fable 5。

Fable 5 的输出价是 Opus 4.8 的 2 倍、Sonnet 4.6 的 3 倍多。这意味着：

  • 值这个价的场景：最难的推理、长链路 agent、一次跑很久的自动化任务——这些地方"一次做对"省下的返工，远比 token 差价值钱。
  • 不必上的场景：日常问答、写作、量大的批处理。Opus 4.8 同样 1M 上下文、价格一半；再往下 Sonnet 4.6 更便宜。盲目全程用 Fable 5，账单会很快失控。
  • 务实做法：用 Sonnet/Opus 当主力，只把真正啃不动的步骤路由给 Fable 5。怎么按任务分层，见 做 AI Agent 用哪个模型。

开发者要注意的三件事

如果你在代码里接 Fable 5（不是 Mythos 5），有三处和以往 Claude 不一样，接入前要规划好：

  • 1. 拒绝（refusal）：Fable 5 的安全分类器拒绝请求时，Messages API 返回的是 stop_reason: "refusal" 的正常 200 响应，不是报错，还会告诉你是哪个分类器拦的。你的代码要把这种情况当一种正常分支来处理，而不是当异常。
  • 2. 兜底（fallback）：被 Fable 5 拒绝的请求，通常可以换一个 Claude 模型重试。官方提供三种方式：服务端（传 fallbacks 参数让 API 自动重试，目前 beta）、客户端（用各语言 SDK 中间件）、或你自己手写重试。
  • 3. 计费：在生成任何输出前就被拒绝的请求不计费；当你换模型重试时，"fallback credit" 会退还切换带来的 prompt cache 成本，避免同一段缓存付两次钱。

另外，发布即支持 effort（控制思考深度）、task budgets（beta）、memory 工具、代码执行、程序化工具调用、上下文编辑、compaction 和视觉理解。

要不要从 Opus 4.8 升级

和上次 Opus 4.7→4.8 那种"同价升级"不同，这次价格翻倍，所以要不要换，得先问"我的任务配得上这个价吗"。按场景：

  • 新项目、且任务确实很难（复杂推理、长程 agent）：可以直接上 Fable 5，趁 6/9–6/22 订阅免费期先压测一轮，看它在你的真实任务上比 Opus 4.8 强多少，再决定长期用不用。
  • 已经在 Opus 4.8 上跑稳的生产/agent 流程：不要急着切。一是价格翻倍，二是 Fable 5 多了 refusal/fallback 这套新行为，贸然换会动到你调好的链路。等你能量化"贵一倍换来多少收益"再说。
  • 日常问答、写作、量大场景：基本没必要。Opus 4.8 甚至 Sonnet 4.6 就够，省下的钱很可观。

一句话：Fable 5 是"按需调用的重器"，不是"默认主力"。它值不值，取决于你有没有真正配得上 $10/$50 的硬任务。

关于跑分

写这篇时，Anthropic 没有公布 Fable 5 的标准 benchmark 具体数字（SWE-bench Verified、Terminal-Bench、GPQA 等都没有给分数）。官方的说法是"在几乎所有测试基准上达到 SOTA"，并点名两项第三方评测：Cognition 的 FrontierCode（称前沿模型中得分最高）、Hebbia 金融基准（称任何模型中最高分）——但都没有给出可对比的数值。

所以这篇不替它编分数。可以拿来当锚点的，是上一代旗舰 Opus 4.7 的公开成绩：SWE-bench Verified 87.6%、SWE-bench Pro 64.3%、Terminal-Bench 69.4%（完整拆解见这里）。等 Fable 5 的官方数据出来，本页会补上它和 Opus 4.8、GPT-5.5 的正面对比。想随时看实时价格和六维能力，用对比工具。
