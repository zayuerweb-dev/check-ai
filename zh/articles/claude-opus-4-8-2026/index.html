已确认的事实

截至 2026-05，关于 Opus 4.8 可以确定的是：

  • 发布日：2026-05-28。
  • 价格：Anthropic 官方 $5 / 1M 输入、$25 / 1M 输出，和 Opus 4.7、4.6 一条线没变。prompt cache 和 batch 折扣按官网。
  • 上下文：1M token。
  • 可用渠道：Anthropic API、AWS Bedrock、Google Vertex、Cloudflare AI Gateway、Vercel、OpenRouter、DigitalOcean 等，主流云和网关基本当天就接上了。
  • 最低价：转售渠道里 Poe 约 $4.29 / $21.46，比官方略低；多数平台跟官方一致 $5 / $25。

注意 GitHub Copilot 那条目前价格显示为"未公开"（它对订阅用户不单独计 token 价），不代表免费。

要不要从 4.7 升级

因为价格和上下文都没变，升级决策比平时简单，按你的处境分三种：

  • 新项目 / 还没绑定版本：直接用 4.8。同价、更新，没理由从头去选上一代。
  • 已经在 4.7 上跑稳的 agent / 生产流程：别急。一个 .x 升级最大的风险不是价格，是行为变化——工具调用的细节、输出风格、边界情况可能微调，会影响你调好的 prompt 和 agent 链路。完整跑分和稳定性反馈还没出来，先观望，等数据 + 社区实测，再小流量灰度切。
  • 只是日常问答 / 写作：切不切都行，体感差别通常不大；想用最新的就切。

一句话：新开的用 4.8，跑着的别动手太快。同价更新的好处是没有成本顾虑，坏处是没有"非升不可"的理由，所以稳定性优先。

Opus 4.8 还是 Sonnet 4.6

比"4.7 还是 4.8"更常见的问题，其实是"要不要用 Opus"。Opus 4.8 是 $5/$25，Claude Sonnet 4.6 最低约 $2.6-3 / $15——Opus 的输出价是 Sonnet 的差不多两倍。

  • 日常、量大、对成本敏感：先用 Sonnet 4.6，够用且便宜不少。
  • 难任务、长链路 agent、复杂推理：这才是 Opus 值回价钱的地方。
  • 务实做法：主力 Sonnet，难的环节路由到 Opus 4.8——别全程烧 Opus。具体怎么按任务分配，见做 AI Agent 用哪个模型。

关于跑分

到本文发布时，Anthropic 还没放出 Opus 4.8 的完整 benchmark（SWE-bench Verified / Pro、Terminal-Bench、GPQA 等），第三方实测也还在early。所以这篇不给 4.8 编任何分数——这是我们的底线：宁可少说，不替模型编数据。

可以参考的基线是 Opus 4.7：SWE-bench Verified 87.6%、SWE-bench Pro 64.3%、Terminal-Bench 69.4%（完整拆解见这篇）。等 4.8 的官方数据出来，我会更新本页并补上和 4.7、GPT-5.5、Gemini 的横向对比。想随时看各模型的实时价格和能力维度，用对比工具。
